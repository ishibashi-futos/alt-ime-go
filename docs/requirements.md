# 要求仕様 (Requirements) — alt-ime

## 1. 目的・背景

alt-ime-ahk（左右 Alt の空打ちで IME を OFF/ON する AutoHotkey スクリプト）相当の機能を、
AutoHotkey ランタイムに依存せず Go でスクラッチ実装する。macOS の「英数／かな」キーのような
明示的な IME 切替体験を Windows で再現し、切替要求の送出結果を邪魔にならない OSD で可視化する。

検証状態と Windows 実機の未検証項目は [todo.md](todo.md) が所有する。

## 2. 用語

| 用語 | 意味 |
|---|---|
| 空打ち | 物理 Alt を押した時点で他キーが down でなく、保持中にも他キーを押さず、規定時間内に離す操作 |
| IME OFF | 直接入力（英数）を要求する状態 |
| IME ON | 日本語入力（かな）を要求する状態 |
| 送出成功 | IME 用 down/up の 2 入力を `SendInput` が入力ストリームへ挿入した、または選択中の IMM32 呼出しが成功したこと |
| 実状態 | 対象アプリ・入力コンテキストが最終的に保持する IME 状態。送出成功とは区別する |
| OSD | 切替要求の送出成功または失敗を短時間表示するインジケータ |
| 自己注入 | 本ツールが `SendInput` し、固有 `dwExtraInfo` マーカーを付けた入力 |
| UIPI | User Interface Privilege Isolation。権限境界によるフック、メッセージ、入力注入の制限 |
| Enter送信ガード | 対象アプリが前面のとき、Enter 単独を Shift+Enter（改行）へ置換し、Ctrl+Enter を素の Enter（送信）へ変換する機能 |
| 対象アプリ | 前面ウィンドウのプロセス exe 名（basename、小文字比較）が定数リストに一致するアプリ |

## 3. 機能要求 (FR)

| ID | 要求 | 優先度 |
|---|---|---|
| FR-1 | 左 Alt の空打ちで、空打ち成立時の前面ウィンドウへ IME OFF を要求する | 必須 |
| FR-2 | 右 Alt の空打ちで、空打ち成立時の前面ウィンドウへ IME ON を要求する | 必須 |
| FR-3 | Alt 押下時または保持中に他キーが down の場合は空打ちを成立させず、Alt+Tab 等を通常動作させる | 必須 |
| FR-4 | 物理 Alt だけをトリガーとする。外部注入キーはトリガーにならないが、進行中の空打ちをキャンセルできる | 必須 |
| FR-5 | 単独 Alt による Win32/Electron のメニューフォーカス移動、および Web アプリの単独 Alt UI を抑制する。Alt 自体はブロックしない | 必須 |
| FR-6 | IME 切替要求の送出成功時に OFF=`A` / ON=`あ` の OSD を表示し、一定時間後にフェードアウトする | 必須 |
| FR-7 | IME 切替要求の送出失敗時は通常 OSD を表示せず、失敗を表す `!` OSD を表示する | 必須 |
| FR-8 | 切替処理前に前面 HWND とトリガー Alt の解放を再検証し、条件不一致なら要求を破棄する | 必須 |
| FR-9 | タスクトレイに常駐し、Explorer 再起動後にアイコンを復旧する | 必須 |
| FR-10 | トレイメニューをマウスとキーボードの両方で操作でき、有効/無効を切り替えられる | 必須 |
| FR-11 | トレイメニューから終了でき、フックと全 UI リソースを順序立てて解放する | 必須 |
| FR-12 | 同一ユーザーセッション内の多重起動を防止する | 必須 |
| FR-13 | OSD の表示位置・不透明度・保持時間・フォント等をコード上の設定値で調整できる | 中 |
| FR-14 | IME 制御方式（VK 送出 / IMM32）を明示的に選択できる。自動二重適用はしない | 中 |
| FR-15 | Alt メニューフォーカス抑制をコード上の設定値で無効化できる | 中 |
| FR-16 | Ctrl+Shift 方式に対応可能な境界を維持する | 低（保留） |
| FR-17 | ログオン時の自動起動に対応する | 低 |
| FR-18 | 設定ファイルで各種パラメータを外出しする | 低 |
| FR-19 | 配布バイナリ名を `alt-ime-go.exe` とし、exe とトレイに共通の墨色×白のアプリアイコンを表示する | 必須 |
| FR-20 | 対象アプリで物理 Enter 単独の down をブロックし、タグ付き Shift+Enter を注入して改行にする（Enter送信ガード） | 必須 |
| FR-21 | 対象アプリで Ctrl+Enter をブロックし、物理押下中の Ctrl の一時解放→Enter→Ctrl 復元をタグ付き注入して素の Enter（送信）として届ける | 必須 |
| FR-22 | Shift/Alt/Win を含む Enter chord、他プロセス注入の Enter、非対象アプリの Enter には介入しない。ブロックした down に対応する up と auto-repeat も対で飲み込む | 必須 |
| FR-23 | トレイメニューで Enter送信ガードを個別に ON/OFF できる | 必須 |
| FR-24 | IME 変換中と推定される Enter（変換中ヒューリスティック成立かつ対象の IME が open）は素の Enter として届け、変換確定を維持する | 必須 |

## 4. 非機能要求 (NFR)

| ID | 要求 |
|---|---|
| NFR-1 | フックを専用 OS スレッドで動かし、callback は固定量・非ブロッキングに近い処理だけを行う |
| NFR-2 | callback 内では UI、GDI、Shell、IMM32、ファイル/標準出力ログ、待機処理を実行しない |
| NFR-3 | 外部依存ゼロ（標準 `syscall` でシステム DLL を直接呼ぶ）。配布は単一 exe |
| NFR-4 | 対応 OS は ImeOn/ImeOff 対応更新以降の Windows 10 1903+ と Windows 11（x64） |
| NFR-5 | GUI サブシステムでビルドし、コンソール窓を出さない（`-H windowsgui`） |
| NFR-6 | 常駐アイドル時 CPU ≒ 0。フェード中以外は周期的ポーリングを行わない |
| NFR-7 | 自己注入は `LLKHF_INJECTED` と `dwExtraInfo` 固有マーカーの両方で識別する |
| NFR-8 | OSD はクリック透過、フォーカス非奪取、最前面、タスクバー非表示とする |
| NFR-9 | process DPI awareness は Per-Monitor V2 とし、混在 DPI でサイズ・位置・フォントを再計算する |
| NFR-10 | Win32 API の失敗を黙殺しない。致命的な初期化失敗は利用者へ表示して安全に終了する |
| NFR-11 | 同期的な外部 HWND 呼出しには `SendMessageTimeout` と有限期限を使う |
| NFR-12 | 空打ち成立から IME 送出開始までの通常時 p95 を 50ms 以下とする |
| NFR-13 | アプリアイコンは 16〜256px の複数サイズを持ち、外部依存なしで再生成・PE 埋め込みできる |

## 5. 制約・既知リスク (CON)

| ID | 制約 |
|---|---|
| CON-1 | 標準権限から管理者アプリへの監視・入力注入・メッセージ送信は UIPI の制約を受ける |
| CON-2 | セキュアデスクトップ（UAC プロンプト、サインイン画面等）は対象外 |
| CON-3 | メニュー抑制は未割当 VK `0x07` と、対象アプリにも `Alt+F24` として届く `VK_F24` の併用。後者はアプリ固有の割り当てと衝突する可能性がある |
| CON-4 | `VK_IME_ON/OFF` を尊重しない IME がある。第三者 IME は検証合格まで対応保証外 |
| CON-5 | OSD は切替要求の送出結果であり、TSF を含む実 IME 状態を確認した表示ではない |
| CON-6 | AltGr レイアウトは初期対応保証外。日本語・US 系キーボードレイアウトを初期対象とする |
| CON-7 | 署名なし exe は SmartScreen/Defender の警告対象になりうる |
| CON-8 | Ctrl+Shift は Windows 標準の入力言語切替と衝突しうる |
| CON-9 | 他プロセスの IME 変換中状態を確実に検出する公式 API がない。Enter送信ガードはキー列からの変換中ヒューリスティック + `IMC_GETOPENSTATUS` で推定し、変換中と推定した Enter は素の Enter として届ける（FR-24）。誤推定は fail-open（アプリ既定の送信動作）に倒すため、マウスクリックで確定した直後の Enter 等は送信になり得る |
| CON-10 | Enter送信ガードの対象判定は exe 名のみ。ブラウザ内の Web チャットはブラウザ全体が巻き込まれるため対象にしない |

## 6. 対象外 (Out of Scope)

- Ctrl+Shift 方式の具体割り当て、Alt 方式との優先順位・排他
- AltGr の合成 Ctrl と物理 Ctrl を区別するヒューリスティック
- 設定 GUI
- macOS / Linux 対応
- IME のカタカナ、半角等の入力モード細分制御
- すべての第三者 IME・ゲーム・リモート環境での動作保証
- UIAccess 署名・安全なインストーラの提供
